## <AnchorLink label="Problem Statement" name />
> Given a list of n integers arr[0..(n-1)], determine the number of different pairs of elements within it which sum to k.

If an integer appears in the list multiple times, each copy is considered to be different; that is, two pairs are considered different if one pair includes at least one array index which the other doesn't, even if they include the same values.


<CodeBlock>{`
int numberOfWays(int[] arr, int k)
`.trim()}</CodeBlock>

### <AnchorLink label="Constraints" name />
1. n is in the range [1, 100,000].
1. Each value arr[i] is in the range [1, 1,000,000,000].
1. k is in the range [1, 1,000,000,000].

### <AnchorLink label="Expected" name />
1. Return the number of different pairs of elements which sum to k.

### <AnchorLink label="Test Cases" name />
1. n = 5, k = 6 and arr = [1, 2, 3, 4, 3], so output = 2
1. n = 5, k = 6 and arr = [1, 5, 3, 3, 3], so output = 4

### <AnchorLink label="Foot Note" name />
1. There's one valid pair 1+5, and three different valid pairs 3+3 (the 3rd and 4th elements, 3rd and 5th elements, and 4th and 5th elements).
